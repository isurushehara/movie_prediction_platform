# Movie Prediction

Movie Prediction is a full-stack movie recommendation system with three parts:

- a FastAPI backend for authentication, movies, ratings, and recommendation APIs
- a Next.js frontend for browsing movies and viewing personalized recommendations
- a separate machine learning pipeline that trains a TF-IDF based recommendation model

## Features

- Browse movies from the backend API
- Search movies in the frontend
- User registration and login with JWT authentication
- Rate movies and receive personalized recommendations
- Generate content-based movie suggestions from the training dataset

## Tech Stack

- Backend: FastAPI, SQLAlchemy, Pydantic, JWT
- Frontend: Next.js, React, TypeScript, Tailwind CSS, Axios
- ML: Python, pandas, scikit-learn, joblib
- Data: CSV movie datasets and generated model artifacts

## Project Structure

- `backend/` - FastAPI application, database models, and API routes
- `frontend/` - Next.js application for the user interface
- `ml-model/` - training and inference scripts for recommendations
- `dataset/` - movie data used for database import and model training

## Prerequisites

- Python 3.10 or newer
- Node.js 18 or newer
- A MySQL or compatible database for the backend

## Backend Setup

1. Open a terminal in `backend/`.
2. Create and activate a virtual environment.
3. Install dependencies:

```bash
pip install -r requirements.txt
```

4. Create a `.env` file in `backend/` with these values:

```env
DATABASE_URL=mysql+pymysql://user:password@localhost:3306/movie_prediction
SECRET_KEY=your-secret-key
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

5. Create the database tables:

```bash
python run.py
```

6. Start the API server:

```bash
uvicorn app.main:app --reload
```

The backend will be available at `http://127.0.0.1:8000`.

## Frontend Setup

1. Open a terminal in `frontend/`.
2. Install dependencies:

```bash
npm install
```

3. Start the development server:

```bash
npm run dev
```

The frontend will be available at `http://localhost:3000`.

The frontend is configured to call the backend at `http://127.0.0.1:8000`.

## ML Model Setup

The recommendation engine uses files generated by `ml-model/train.py` and loaded by `ml-model/recommend.py`.

1. Open a terminal in `ml-model/`.
2. Install the required packages listed in `ml-model/requirements.txt`.
3. Run the training script to generate the model artifacts:

```bash
python train.py
```

This creates the model files used by the recommendation endpoint.

## Useful Endpoints

- `GET /` - API health message
- `GET /movies` - list movies
- `GET /movies/{movie_id}` - get a single movie
- `GET /movies/search/?title=...` - search movies by title
- `GET /movies/top-rated` - top rated movies
- `GET /recommend/{movie_name}` - similar movie recommendations

## Notes

- The frontend expects the backend to be running before you open the app.
- The recommendation API requires the trained model files in `ml-model/models/`.
- The backend CORS configuration allows requests from `http://localhost:3000`.
